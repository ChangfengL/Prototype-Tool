# 原型标注工具（Prototype Annotation Tool）

> 一个在浏览器里给任意 HTML 页面做"可视化点选标注"的评审工具。
> 本 README 用于让**新会话/接手者**快速理解它的来龙去脉与当前状态。

---

## 一、这是什么

让评审者（哪怕不懂代码）能直接在浏览器里：

**打开页面 → 点中某个元素 → 写"这里要怎么改" → 自动打编号针脚 → 一键导出标注清单（按板块分组的 Markdown）→ 粘贴给 AI，由 AI 按标注精确改代码。**

目标：把口头/截图描述的修改意见，变成**精确定位到具体元素**的结构化清单，大幅提高原型（HTML）迭代效率。

---

## 二、背景：为什么是现在这个形态

1. **最初**：标注功能是**内嵌在原型 HTML 文件里**的（直接往页面塞 CSS/HTML/JS）。
2. **暴露的问题**：① 污染原型文件、体积变大；② 退出时清理不干净有残留风险；③ 每次注入/移除都改文件字节，**快照和 diff 变脏**。
3. **决策**：把标注逻辑**彻底外置**，在浏览器侧运行时注入浮层，**永不写进被标注的文件**。
4. **架构对比结论（书签 vs 浏览器扩展）**：
   - **书签 bookmarklet**：零安装、零污染、file:// 可用；但能力有上限（无截图、无原生侧边栏、持久化较弱）。
   - **浏览器扩展 extension**：能力最强（截图、原生侧边栏、持久化、精确导出、快捷键），但需要安装一次。
   - **选定路径**：先做**书签热身版 v0** → 手感满意后升级**浏览器扩展 v1** → （可选）打包成团队 Skill。
   - 使用者主要在**真实 Chrome/Edge** 里查看页面（不是嵌入式预览）。

## 三、当前状态：书签版 v0（已完成）

- **本目录文件**：
  - `原型标注工具.html` —— 同时是**安装页**（拖书签 / 复制书签代码）和**源码载体**。
    工具源码放在页面里的 `<script type="text/plain" id="anno-src">` 中，由页面脚本用 `encodeURIComponent` 生成 bookmarklet（绕开 HTML 属性转义问题）。
  - `README.md` —— 本文件。
- **已实现功能**：
  - 浮动启动按钮（**可拖动**）、标注模式开关
  - 鼠标悬停高亮、点击元素弹框输入修改意见、自动打**编号红色针脚**
  - 针脚锚定到元素（滚动/缩放时**重新定位**）
  - 标注清单面板：**可拖动 + 可折叠**；按"板块"**分组**展示；可编辑/删除/清空
  - 底部**自动消失的 toast** 提示（点击穿透、不挡选取）
  - 一键"📋 复制给AI"：导出**按板块分组的 Markdown**
  - 标注存浏览器 `localStorage`（按 `location.pathname` 分库），刷新不丢

### 已解决的 5 个评审优化点
1. 启动按钮位置写死、滚动难点 → 浮动**可拖动**按钮
2. 顶部红色提示挡元素 → **自动消失 toast** + 点击穿透
3. 标注清单写死挡元素 → 面板**可拖动 + 可折叠**
4. 清单可读性差 → 导出**按板块分组**、字段规整
5. 嵌在页面里影响文件 → **彻底外置**为书签，零污染

### v0.1 视觉与交互打磨（「REDLINE · 制图红笔」改版）

1. 启动按钮做成**真·开关**（滑轨+滑块+`ON/OFF`），明确表达启用状态。
2. 落地页、开关、清单、提示、输入框**整体视觉重做**为 REDLINE 风格。
3. 浮窗拖动**钳制在视口内**（留 6px 边距），resize 时对已拖动的浮窗重新钳制——不会被拖出可视区。
4. 开启提示移到**居中偏上黄金区（top:20vh）**，放大、加图标+主副文案、从上方滑入。
5. 标记输入由 `prompt()` 升级为**可拖动的页内浮层**：多行 `textarea`、显示所标元素板块+摘要、`Ctrl/⌘+Enter` 保存 / `Esc` 取消、出现在点击点旁并钳制在视口内。

## 四、怎么用

1. 用 Chrome/Edge 打开 `原型标注工具.html` → 把「📍 原型标注」按钮**拖到书签栏**（或"复制书签代码"后手动新建书签把代码粘进网址）。
2. 打开任意要标注的页面（拖入或本地服务器均可，file:// 也行）→ 点书签 → 进入标注模式。
3. 点页面元素写批注；拖动面板/按钮避让遮挡；标完点「📋 复制给AI」→ 粘贴给 AI。

---

## 五、导出格式（接手的 AI/会话据此识别）

用户粘贴过来的标注清单形如：

```
【原型标注清单】共 N 条

## <板块名 / 上下文>
- [序号] <修改说明>
    · 元素："<元素文本摘要>" · 选择器：<CSS 选择器>
```

> 早期版本格式略有差异，但都包含：**序号 + 板块/上下文 + 元素摘要 + CSS 选择器 + 修改说明**。
> AI 拿到后据此**定位元素并改代码**（注意：选择器是辅助定位，最终应结合元素文本与上下文精确匹配）。

## 六、玩法路线图（Backlog，按价值簇组织）

> 核心价值循环：**「人指出哪里、要改成什么」→ 结构化 → AI 精确改代码**。下列玩法按"强化循环的哪一段"分簇。
> 标记：🔖 = 书签 v0 即可做（不必等扩展）；🧩 = 必须浏览器扩展才能稳妥实现。
> **已交付（v0.2 / v0.3）：簇 1 的「可视化微调（含改文案）」「类型感知」「两模式分离」。** 见下方 ✅。

### 簇 1 · 让"指出意图"更精确（capture）——最高杠杆
把模糊文字描述变成精确数值/动作，是"精确元素调整"的正解。
- ✅🔖 **可视化直接微调（所见即所得）**：选中元素弹微调台，改字号/字重/颜色/背景/圆角/宽高，**实时预览**；导出 `font-size 13px → 30px` 等 before→after 精确值。【v0.2/v0.3】
- ✅🔖 **改文案**：纯文本元素可直接改文字，导出"文案 A→B"。【v0.2】
- ✅🔖 **元素类型感知**：识别 文字/按钮/图片/输入/容器，按类型只显示相关控件，弹窗与导出均标注类型。【v0.3】
- ✅🔖 **两模式分离**：一条标注只能是「🎛 直接微调」或「✍️ 文字说明」之一，消除"以哪个为准"的来源冲突。【v0.3】
- ✅🔖 **标记升级**：从浮在上方的大圆点改为「**元素细外框 + 左上角小角标**」，精准框出被改元素；hover 角标 / 清单条目互相高亮，点清单条目**定位回显**（滚动到元素 + 闪烁）。【v0.4】
- ✅🔖 **开关与清单合并**：启动开关与标注清单合为**一个可整体拖动的浮窗**（OFF/空时收缩为开关）。【v0.4】
- 🔖 **分类标签**：改 / 删 / 加 / 移动 / 换色 / 提问，导出分组（"提问"类不改代码、只抛给开发）。【待做】
- 🔖 **拖拽改位置/间距**：拖句柄直接改尺寸/位置/margin。【待做】
- 🧩 **框选区域批注 + 截图**：圈一块区域（不限单元素）配图，让 AI 精确看到指哪。

### 簇 2 · 让"改代码"更自动（close the loop）
- 🔖 **更稳的元素定位**：选择器 + `outerHTML` 片段 + 文本锚点多重冗余，AI 100% 命中。
- 🔖 **设计 token 感知**：识别页面用的 CSS 变量 / Tailwind 类，导出用语义（`--primary`、`text-lg`）而非硬编码值。
- 🧩 **一键直连 AI / 写回文件**：扩展内置调用本地 Claude 或 API，改完直接落到文件，省掉复制粘贴。
- 🧩 **改动 diff 预览**：AI 改完后新旧叠加高亮，评审者点"接受/拒绝"，形成闭环。

### 簇 3 · 从"被动记录"到"主动走查"（proactive）
- 🔖 **设计规范走查**：扫描 DOM，自动标出不合规处（间距非 8 的倍数、字号不在色阶、颜色不在 token 表、对比度不达 a11y）。
- 🔖 **响应式检查**：一键切换视口断点，标注在哪个尺寸下溢出/错位。
- 🔖 **内容体检**：占位文案(lorem)、图片缺失、文本截断。

### 簇 4 · 协作与流程（collaboration）
- 🧩 **多人多角色标注**：PM/设计/前端各自批注、颜色区分、合并。
- 🧩 **评论线程**：每个针脚可回复讨论，像 Figma 评论。
- 🔖 **版本对比**：迭代到下一版，自动把上一版"已解决"的标注标灰。
- 🔖/🧩 **多格式导出**：给 AI 的 Markdown、给设计的截图标注版、给前端的 issue 列表。
- 🧩 **飞书打通**：一键把清单发群 / 建多维表格 / 建任务。

### 簇 5 · 沉淀与度量
- 🔖 **标注统计**：哪个板块改得最多、迭代几轮。
- 🔖 **标注回灌 PRD**：评审意见沉淀成需求记录。

### 浏览器扩展 v1（承载 🧩 项的载体）
扩展是上面所有 🧩 能力的前提：截图/框选、原生侧边栏+工具栏图标、按 URL 分库持久化、直连 API 写回文件、多人协作、快捷键。
最终可打包成 **Skill**：维护扩展源码 + 安装指引 + 导出格式约定。

## 七、技术要点（维护者备查）

- **bookmarklet 生成**：源码放 `<script type="text/plain" id="anno-src">`，页面脚本读 `textContent` → `'javascript:' + encodeURIComponent('(function(){' + src + '\n})();')`，写入书签 `href` 与复制框。
- **幂等**：全局开关 `window.__PROTO_ANNO__`；已初始化则再次点击书签 = 切换标注模式。
- **选择器生成**：优先 `#id`；否则向上构造 `tagName:nth-of-type(n)` 路径（≤5 层）。
- **板块识别**：就近 `[data-board]` → `.panel/.kpi-card/section/header/footer` 的标题文本 → 否则"页面"。
- **持久化键**：`proto-anno::<location.pathname>`。
- **改源码后**：需**重新生成书签**（重新拖/复制），旧书签不会自动更新。
- **依赖**：无外部依赖；纯原生 JS；不联网。
